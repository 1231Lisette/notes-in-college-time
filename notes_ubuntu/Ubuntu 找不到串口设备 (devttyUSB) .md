# 📝 Ubuntu 找不到串口设备 (/dev/ttyUSB*) 踩坑笔记

## 1. 问题现象

在 Ubuntu 系统中插入 CH340/FT232 等 USB 转串口模块后，系统底层的硬件能识别，但无法生成串口设备节点，导致 ROS 2 节点或串口助手无法通信。

具体表现为：

- ❌ `ls /dev/ttyUSB*` 提示：`没有那个文件或目录`
- ✅ `lsusb` 能正常看到设备（如：`QinHeng Electronics CH340 serial converter`）
- ✅ `lsmod | grep ch34` 显示驱动（`ch341`）已正常加载

## 2. 根本原因 (Root Cause)

罪魁祸首是 Ubuntu 预装的一个名为 **`brltty`** 的后台服务。

- **`brltty` 是什么**：这是一个为视障人士提供的盲文显示器终端服务。
- **冲突原因**：由于市面上许多盲文显示器恰好也使用了 CH340 系列的串口转换芯片，当 `brltty` 服务在后台运行时，只要检测到 CH340 接入，就会**自动且强制地接管该设备**，导致系统的标准串口驱动无法为其生成 `/dev/ttyUSB0` 节点。

## 3. 解决步骤

彻底解决此问题只需卸载该冲突服务并重新分配权限：

### Step 1: 彻底卸载 brltty 服务

打开终端，执行以下命令将该服务及其配置文件彻底移除：

```
sudo apt-get remove --purge brltty
```

### Step 2: 重新插拔设备

将你的 USB 转串口模块从电脑上**拔下，然后重新插入**。这一步是为了让系统内核重新接管该硬件并分配资源。

### Step 3: 验证设备是否出现

再次检查设备节点：

```
ls /dev/ttyUSB*
```

此时终端应正常输出 `/dev/ttyUSB0`（或相应的编号）。

### Step 4: 赋予串口读写权限（关键）

设备出现后，默认只有 `root` 和 `dialout` 用户组有权限读写。为了让 ROS 2 节点能正常打开串口，需要赋予权限：

```
sudo chmod 777 /dev/ttyUSB0
```
