| 特征         | 进程 (Process)                                          | 线程 (Thread)                                             |
| :----------- | :------------------------------------------------------ | :-------------------------------------------------------- |
| **基本概念** | 正在运行的程序，是系统资源分配和调度的基本单位。        | 进程中的一个执行单元，是CPU调度的基本单位。               |
| **资源**     | 拥有独立的虚拟内存空间、代码、数据和系统资源。          | 共享所属进程的内存空间和系统资源，但有各自的寄存器和栈。  |
| **关系**     | 一个进程可以包含多个线程。                              | 是进程的子任务。                                          |
| **通信**     | 进程间通信（IPC）相对复杂，需要通过操作系统提供的机制。 | 线程间通信相对简单，可以直接通过共享内存进行。            |
| **开销**     | 创建和销毁进程的开销较大，涉及资源分配和回收。          | 创建和销毁线程的开销较小，只需保存寄存器和栈信息。        |
| **独立性**   | 进程之间相互独立，一个进程崩溃通常不会影响其他进程。    | 线程之间相互依赖，一个线程崩溃可能影响整个进程的稳定性。  |
| **执行**     | 多个进程可以并发执行。                                  | 在同一进程中，多个线程可以并发或并行（在多核CPU上）执行。 |
